PROJECT DESCRIPTION

Auto Pasture is a web application that allows users to search used and new vehicles by make model and year based on their location. It also allows users to like and follow particular vehicles that they may be interested in. Built backend with Ruby on Rails and utilized ActiveRecord with a many to many SQL database relationship. Frontend UI was built with React and Redux, while heavily leaning on Semantic-UI-React and Thunk Middleware for asynchronous 'effect' of API calling.

  • Used geolocation to prompt user in gaining permission to use their Latitude and Longitude coordinates.
  • Sign-up uses a second password for confirmation otherwise prompting the user an error
  • Used the bcrypt, dotenv-rails, and jwt gems to establish secure authentication and authorization
Every vehicle card contains links to like/follow, see images within a modal, connect you to google maps with vehicles exact location, and to the seller's website. Most of the API data was complete, however, I found it necessary to implement a catch and fall back for every vehicle parameter that may query with empty content or as undefined.
When planning this project, I specifically wanted an API that provided me vehicle data through an API call rather than juggle large JSON data sets that would sit within the directories.
Users can search for a specific vehicle and receive relevant cost and milage data of other similar cars within their desired milage radius.
  • Render unknown and complex vehicle data from user selected dropdown forms.
  • Data is graphed in an easy to understand visual data-plot by cost and milage by using the Recharts library.
  • Every data-plot shows the minimum, maximum, national mean, and national average vehicle.
  • Comprehensively utilized the Marketcheck API by implementing a variety of unique API calls depending on user input data.
  • Coupled Thunk Middleware with a javascript Promise allowing me to iterate over an array of data while simultaneously calling the API.
  • Each selected dropdown makes an API call that then uses the response to fill the next dropdown with relative and accurate data. This prohibits any user from searching vehicles that do not exist or are not currently being sold.
